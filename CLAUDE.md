## Architecture

ReliaLearnR is an educational R package with two distinct layers:

**1. Utility functions** (`R/helpers.R`) — Five RAM calculation helpers:
[`rel()`](https://paulgovan.github.io/ReliaLearnR/reference/rel.md),
[`avail()`](https://paulgovan.github.io/ReliaLearnR/reference/avail.md),
[`mttf()`](https://paulgovan.github.io/ReliaLearnR/reference/mttf.md),
[`mtbf()`](https://paulgovan.github.io/ReliaLearnR/reference/mtbf.md),
[`fr()`](https://paulgovan.github.io/ReliaLearnR/reference/fr.md). These
are thin numeric computations with input validation. All validation
follows the pattern: check NA → check type → check non-negative → check
non-zero denominator.

**2. Tutorial launchers** (`R/ram.R`, `R/lda.R`, `R/rt.R`) — Thin
wrappers around
[`learnr::run_tutorial()`](https://pkgs.rstudio.com/learnr/reference/run_tutorial.html)
that launch interactive Shiny-prerendered tutorials stored in
`inst/tutorials/{ram,lda,rt}/`. The tutorials depend on `WeibullR`,
`WeibullR.ALT`, and `ReliaGrowR` for analysis content.

**Documentation** is entirely roxygen2-generated — never edit files in
`man/` or `NAMESPACE` directly. Run `devtools::document()` after
changing roxygen tags.

**Tests** live in `tests/testthat/`: `test-helpers.R` covers the utility
functions (89+ cases including input validation, edge cases, and vector
handling); `test-launchers.R` covers the tutorial launchers
(interactive-only functions).

The `docs/` directory is generated by pkgdown and is git-ignored; `man/`
is generated by roxygen2 and is committed.
